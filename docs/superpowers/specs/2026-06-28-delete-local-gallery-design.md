# 删除本地图库单本设计

## 目标

允许用户在 Web 面板的“本地图库”页面删除单本已下载画廊，并同时清理：

- 本地图片目录
- 数据库中的 `galleries` 记录
- 通过外键级联关联删除的 `gallery_tags`

本次范围仅包含“删单本”，不包含批量删除和清空整个本地库。

## 现状

- `ehlib/storage/file_manager.py` 已有 `delete_gallery_dir()`，可删除单本本地目录。
- `ehlib/models/database.py` 已有 `delete_gallery()`，可按 `source` + `source_id` 删除数据库记录。
- `web/index.php` 的“本地图库”仅支持查看和筛选，不支持删除。
- `web/api.php` 尚无删除单本画廊的 API。

## 方案

### 后端

在 `web/api.php` 新增 `delete_gallery` 动作：

- 输入参数：`source`、`source_id`
- 处理流程：
- 校验参数非空
- 查询数据库获取对应画廊记录及 `local_path`
- 若记录不存在则返回错误
- 若 `local_path` 存在且位于项目下载目录内，则删除该目录
- 删除数据库记录
- 返回删除结果与简要消息

### 前端

在“本地图库”表格增加“操作”列，并在每一行显示 `删除` 按钮：

- 点击按钮时阻止行展开事件冒泡
- 弹出二次确认
- 确认文案明确提示会删除本地文件和数据库记录
- 删除成功后显示 toast，并刷新当前列表
- 删除失败后显示错误 toast

## 安全边界

- 删除目标由数据库记录确定，不接受前端传入任意文件路径。
- 仅删除位于项目下载目录下的本地目录，避免误删库外路径。
- 删除失败时不得继续静默返回成功。

## 错误处理

- 画廊不存在：返回 `Gallery not found`
- 本地目录删除失败：返回具体错误
- 数据库删除失败：返回具体错误
- 若本地目录已不存在，但数据库记录存在，允许继续删除数据库记录

## 验证

- 在本地图库删除一条 `nhentai` 记录，确认目录消失、数据库记录消失、页面刷新后不再显示
- 在本地图库删除一条 `exhentai` 记录，确认行为一致
- 删除不存在的记录时，前端能收到并显示错误
